# Kink Questionnaire Comparison Tool

A simple React/Vite application that compares two shared questionnaire profiles and calculates a compatibility score based on their responses.

The application accepts two share URLs generated by the Kink Questionnaire application, decodes the profile data, compares responses, and displays an overall compatibility percentage along with a question-by-question breakdown.

---

## Features

- Paste two questionnaire share URLs
- Decode profile data directly in the browser
- Calculate compatibility score
- Display side-by-side response comparison
- No backend required
- No external database required
- Fully client-side
- Built with React and Vite

---

## Compatibility Calculation

Each answer is expected to be rated on a scale from:

| Value | Meaning |
|---------|---------|
| 1 | Hard Limit |
| 2 | Not Interested |
| 3 | Neutral |
| 4 | Interested |
| 5 | Very Interested |

For each question:

1. Calculate the difference between Partner A and Partner B.
2. A difference of 0 receives 100%.
3. A difference of 4 receives 0%.
4. The average score across all questions becomes the overall compatibility percentage.

Examples:

| A | B | Compatibility |
|---|---|---|
| 5 | 5 | 100% |
| 5 | 4 | 75% |
| 5 | 3 | 50% |
| 5 | 2 | 25% |
| 5 | 1 | 0% |

## Installation

Clone the repository:

```bash
git clone <repository-url>
cd compare-questionnaire
```

Install dependencies:

```bash
npm install
```

Start development server:

```bash
npm run dev
```

The application will typically be available at:

```text
http://localhost:5173
```

## Input Format

The application expects URLs generated by the questionnaire application.

Example:

```text
https://example.com/?data=eyJ2ZXJzaW9uIjoiMi4wIiwibmFtZSI6IlNjb3R0IiwiYW5zd2VycyI6eyJxMCI6NSwicTEiOjN9fQ==
```

The `data` parameter contains a Base64 encoded JSON object.

Example decoded structure:

```json
{
  "version": "2.0",
  "name": "Scott",
  "answers": {
    "q0": 5,
    "q1": 3,
    "q2": 4
  }
}
```

---

## Expected Questionnaire Format

Question definitions are stored separately from the profile data.

Example:

```javascript
const questionSets = {
  "2.0": [
    "Question 1",
    "Question 2",
    "Question 3"
  ]
};

export default questionSets;
```

The profile's version number determines which question set should be used during comparison.

## Security Notes

All processing occurs entirely within the user's browser.

The application:

- Does not send questionnaire data to a server
- Does not store profile information remotely
- Does not require authentication
- Does not require a database

Data remains local to the user's device unless they manually share questionnaire URLs.
